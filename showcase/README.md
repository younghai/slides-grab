# slides-grab Showcase

A static gallery of presentations built with [slides-grab](https://github.com/vkehfdl1/slides-grab). Auto-deployed to GitHub Pages on every push to `main`.

## Layout

```
showcase/
├── index.html                    Gallery landing page
├── assets/
│   ├── css/styles.css            Dark minimal theme
│   ├── js/gallery.js             Manifest loader + card renderer
│   └── manifest.json             Generated by build-manifest.js
├── scripts/
│   └── build-manifest.js         Scans presentations/ and emits manifest.json
└── presentations/
    └── <slug>/                   One folder per deck (see "Adding a deck")
```

## Adding a deck

1. Create a folder under `showcase/presentations/`. The folder name becomes the URL slug.
   ```
   showcase/presentations/my-deck/
   ```
2. Copy a slides-grab deck into it. The minimum required is one or more `slide-XX.html` files. Recommended:
   - `viewer.html` — the standalone interactive viewer (run `slides-grab build-viewer --slides-dir <path>`)
   - `slide-outline.md` — used to auto-extract the title (first H1)
   - `preview-png/slide-01.png` — used as the gallery thumbnail (run `slides-grab png --slides-dir <path>`)
   - `assets/` — any local images/videos referenced by slides
   - A `*.pdf` file at the deck root — surfaced as a download link

3. (Optional) Add `meta.json` to override auto-detected fields:
   ```json
   {
     "title": "Custom title",
     "description": "One-line tagline shown under the title",
     "tags": ["research", "ml"],
     "date": "2024-12-01",
     "thumbnail": "preview-png/slide-01.png",
     "pdf": "deck.pdf",
     "hidden": false
   }
   ```
   Set `"hidden": true` to keep a deck on disk without listing it in the gallery.

4. Rebuild the manifest:
   ```bash
   npm run build:showcase
   ```
   Then open `showcase/index.html` in a browser to verify locally.

## Running locally

The site is fully static — no dev server is required, but a local server avoids `file://` CORS quirks for `manifest.json`:

```bash
npm run build:showcase
npx serve showcase            # or: python3 -m http.server -d showcase 8000
```

## Deployment

Pushes to `main` that touch `showcase/**` trigger `.github/workflows/pages.yml`, which:

1. Runs `node showcase/scripts/build-manifest.js`
2. Drops a `.nojekyll` file
3. Uploads `showcase/` as a Pages artifact
4. Deploys via `actions/deploy-pages@v4`

**One-time setup** (must be done in the repo's GitHub UI):
**Settings → Pages → Source → GitHub Actions**.

## Excluding decks from the public site

Two ways to keep a deck on disk without publishing it:

- Add `"hidden": true` to the deck's `meta.json` (recommended — keeps the directory but skips it from the manifest).
- Don't copy the deck into `showcase/presentations/` at all.

`showcase/presentations/` is not gitignored, so anything you put there will be committed and deployed. Only place decks here that are safe to publish.
